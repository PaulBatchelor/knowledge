---
okay looking at the zero page task. lots of rambling there
---
I think this is what the blockstack improvements are:
changing the data to use packed 10-bit numbers, and
creating a way to turn those numbers to memory addresses.
---
the problem of packing 10-bit ints into 32-bit word memory
is tricky due to alignment. I'm wondering if it might
be beneficial to generalize this problem. It turns out
that the entire memory can be addressable as a bit array.
Sonilo memory has $2^16$ 32-bit words, which means
that there are $2^16 2^5 = 2^21$ addressable bits, which
fits inside of a 32-bit number.
---
There needs to be a way to convert blockstack addresses
to word addresses. It's a kind of linear mapping, but it
skips the range of addresses needed for the block stack.
---
The 'ba' command returns block addresses, which are just
those 10 bit numbers. Another utility factoring, in the
block stack address should be used for conversion.
Not sure what that looks like though.
